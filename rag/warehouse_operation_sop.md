# 창고 운영 예외 대응 SOP

## 재고 부족 예상 대응
1. 입고예정 데이터 확인
2. 7일 이내 입고예정 여부 확인
3. 입고예정이 없으면 긴급 보충 요청 검토
4. 우선순위 높은 출고 주문부터 재고 배정
5. 대체 SKU 가능 여부 확인

## CAPA 부족 대응
1. 동일 보관조건 Zone의 여유 CAPA 확인
2. 저회전 SKU 재배치 가능 여부 확인
3. 임시 보관 Location 확인
4. 입고 일정 조정 검토
5. 관리자 승인 요청

## 보관조건 불일치 대응
1. 후보 Location에서 제외
2. 동일 보관조건 Zone 재검색
3. 적합 Zone이 없으면 예외 알림
4. 관리자 확인 요청

## 적재 가능 Location 없음 대응
1. 보관조건이 일치하는 전체 Zone의 Location 재검색
2. 후보가 없는 사유 확인 (CAPA 부족, 사용 불가 Location, 보관조건 불일치)
3. CAPA 부족이 사유이면 CAPA 부족 대응 절차 수행
4. 임시 보관 Location 또는 입고 Dock 대기 처리 검토
5. 관리자 승인 요청

## 출고시간 임박 대응
1. 긴급 피킹 대상으로 표시
2. 피킹지시 우선 발행 권고
3. 작업자 배정 가능 여부 확인
4. 지연 예상 시 관리자 알림

## 피킹지시 미발행 대응
1. 출고예정 주문 중 피킹지시 미발행 건 조회
2. 권장 피킹 시작시간이 도래했거나 임박한 주문 우선 표시
3. 재고 가용 여부 확인 후 피킹지시 Draft 생성
4. 사용자 승인 후 피킹지시 발행
5. 시작시간이 이미 지난 주문은 긴급 피킹으로 분류하고 관리자 알림

## 입고 지연 대응
1. 입고예정일이 지난 미입고 건 조회
2. 해당 SKU의 재고 리스크 영향 확인 (예상소진일 재계산)
3. 재고 부족 위험이 있으면 재고 부족 예상 대응 절차 수행
4. 공급처 입고 일정 재확인 요청
5. 입고예정일 변경 시 Forecast 재계산

## 출고확정 지연 대응
1. 피킹완료 여부 확인
2. 패킹완료 여부 확인
3. 출고확정대기 목록 상단 표시
4. 사용자 승인 후 출고확정 처리
